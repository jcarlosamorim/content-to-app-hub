# Método NEXORAMA

> O Método de Transformação de Conteúdo em Experiência

**Como pegar CAOS (fala, texto, vídeo) e transformar em SISTEMA EXECUTÁVEL**

## Os 3 Problemas Resolvidos

### 1. Dados Escuros
| Contexto | Manifestação |
|----------|--------------|
| Técnico | Código legado, reuniões não documentadas |
| Humano | Conhecimento na cabeça do especialista nunca extraído |

### 2. Ambiguidade
| Contexto | Manifestação |
|----------|--------------|
| Técnico | "Processe os dados" (quais dados?) |
| Humano | "Seja mais produtivo" (como exatamente?) |

### 3. Gap Intenção → Execução
| Contexto | Manifestação |
|----------|--------------|
| Técnico | Cliente quer X, código faz Y |
| Humano | Pessoa quer mudar, não sabe os passos |

---

## Insight Central

> Em vez de dar PDF com framework para preencher, criar **EXPERIÊNCIA INTERATIVA** que EXTRAI o que preciso do usuário enquanto ele usa.

---

## As 4 Fases

### Fase 1: INGESTÃO

| Campo | Valor |
|-------|-------|
| Tempo | 5-30 min |
| Input | Reels, vídeo YouTube, livro, podcast, framework |

**Processo:**
1. Transcrever (se áudio/vídeo)
2. Identificar o SISTEMA LATENTE — a lógica operacional escondida

**Pergunta-Chave:**
> "Qual é o FLUXO DE FUNCIONAMENTO que o autor está descrevendo, mesmo que não esteja explícito?"

---

### Fase 2: DECOMPOSIÇÃO AOC

| Campo | Valor |
|-------|-------|
| Tempo | 15-60 min |
| Output | Lista estruturada de instruções decompostas |

**Processo:**
1. Identificar todas as AÇÕES recomendadas
2. Decompor cada ação em AOC (Ação + Objeto + Condição)
3. Mapear DEPENDÊNCIAS entre ações
4. Identificar VARIÁVEIS do usuário

**Estrutura AOC:**
| Elemento | Pergunta |
|----------|----------|
| Condição | Quando/Se? (gatilho) |
| Ação | Faça o quê? (verbo) |
| Objeto | Como/O quê especificamente? (detalhamento) |
| Resultado | Para quê? (benefício) |

**Exemplo:**

*Frase original:*
> "Você precisa priorizar as tarefas mais importantes logo de manhã quando sua energia está alta"

*Decomposição:*
| Elemento | Valor |
|----------|-------|
| Condição | Manhã / Energia alta |
| Ação | Priorizar |
| Objeto | Tarefas mais importantes |
| Resultado | Maior produtividade |

---

### Fase 3: ARQUITETURA DO NEXO APP

| Campo | Valor |
|-------|-------|
| Tempo | 10-30 min |
| Output | Grafo de decisão |

**Perguntas-Chave:**
1. Quais PERGUNTAS preciso fazer ao usuário?
2. Em que ORDEM?
3. Quais são os CAMINHOS possíveis?
4. Onde estão os PONTOS DE PERSONALIZAÇÃO?

---

### Fase 4: CONSTRUÇÃO DO NEXO APP

| Campo | Valor |
|-------|-------|
| Tempo | 5-60 min |
| Output | Nexo App funcional com protocolo personalizado |

**Princípios de Design:**
1. Cada tela = 1 decisão/informação
2. Mostrar VALOR a cada passo
3. Usar visuais quando possível
4. IA para CLARIFICAR, não ADIVINHAR
5. Loop de refinamento: mostrar → validar → ajustar

## Prompt Master de Extração

```markdown
# PROMPT NEXORAMA

Analise este conteúdo e extraia:

## 1. SISTEMA CENTRAL
Qual é a lógica operacional principal?

## 2. DECOMPOSIÇÃO AOC
Liste todas as ações no formato:
- CONDIÇÃO: (quando/se)
- AÇÃO: (faça)
- OBJETO: (o quê/como)
- RESULTADO: (para quê)

## 3. VARIÁVEIS DO USUÁRIO
O que muda de pessoa para pessoa?

## 4. GRAFO DE DECISÃO
Em que ordem perguntar para personalizar?

## 5. SISTEMA DE CAPTAÇÃO
Que tipo de Captador funcionaria melhor?
- Afunilante / Socrático / Temporal / Espelho

[COLE O CONTEÚDO AQUI]
```

## Glossário

| Termo | Definição |
|-------|-----------|
| **Nexorama** | O processo completo de transformar conteúdo em Nexo App |
| **Nexo App** | O produto final — experiência interativa que personaliza conhecimento |
| **Decomposição AOC** | Análise fundamental: Ação + Objeto + Condição |
| **Sistema de Captação** | Os 4 padrões de UI para extrair contexto |
| **Captador Afunilante** | Padrão que estreita opções progressivamente |
| **Captador Socrático** | Padrão que revela o que o usuário não sabia |
| **Captador Temporal** | Padrão que mapeia no tempo |
| **Captador Espelho** | Padrão que compara com modelo |
